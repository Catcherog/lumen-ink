## 一、核心优势：单平台一体化

### 1.1 为什么这是最佳方案？

**关键信息**：你确认"Shotlab上面可以使用即梦的模型"

这意味着：

✅ **无需跨平台操作**
- 不需要在Shotlab和即梦网站之间切换
- 所有操作在一个界面内完成
- 文件传输更便捷，效率更高

✅ **统一的工作环境**
- 同一个画布中可以混合使用不同模型
- 数据流转更顺畅，格式兼容性更好
- 操作习惯一致，学习成本更低

✅ **简化后的流程**
- 原本设计的"Shotlab生成 + 即梦修复"双平台流程
- 现在可以简化为"Shotlab内全流程"
- 减少了平台切换带来的复杂度和潜在问题

### 1.2 与原方案的对比

| 维度 | 原方案（双平台） | 新方案（Shotlab一体化） |
|------|----------------|----------------------|
| **平台数量** | 2个（Shotlab + 即梦） | **1个（Shotlab）** ⭐ |
| **文件传输** | 需要导出/导入 | **内部节点连接** ⭐ |
| **操作连贯性** | 中断（需切换网站） | **完全连续** ⭐ |
| **学习成本** | 需学两套界面 | **只需学一套** ⭐ |
| **数据管理** | 分散在两处 | **集中管理** ⭐ |
| **潜在风险** | 格式不兼容、版本错乱 | **风险极低** ⭐ |

### 2.2 核心设计原则

#### **原则1：模块化与可选性**
- Phase 1是必须的（基础生成）
- Phase 2是按需的（只在有问题时才执行）
- Phase 3和4是完全可选的高级功能
- 你在每个质检点决定是否进入下一阶段

#### **原则2：人工确认机制**
- 符合你"半自动模式"的偏好
- 关键节点后都有你的确认环节
- 保证质量可控，避免AI出错无法挽回

#### **原则3：模型分工明确**
- **Shotlab原生模型**（Image2/Banana）：负责"从无到有"的生成
- **即梦模型**：负责"从有到优"的修复和精修
- 各自发挥所长，避免功能重叠或冲突

## 四、Phase 2：后期修复（Shotlab内的即梦模型）

> ⚠️ **重要说明**：此阶段的详细步骤依赖于你在**任务1.2**中探索到的即梦功能具体情况。以下是基于通用即梦功能的预设方案，实际执行时可能需要根据Shotlab的实际界面进行调整。

### 4.1 问题诊断与方案匹配

**常见问题类型及对应的即梦解决方案**：

| 问题类型 | 典型表现 | 推荐的即梦功能 | 复杂度 |
|---------|---------|--------------|--------|
| **多余元素** | 有路人、杂物、穿帮镜头 | 消除画笔 | ⭐ 简单 |
| **小瑕疵** | 面部崩坏、文字乱码、纹理错误 | 细节修复 | ⭐ 简单 |
| **构图问题** | 比例不对、主体位置差 | 扩图 + 抠图 | ⭐⭐ 中等 |
| **多元素拼接** | 需要添加新元素或重组构图 | 抠图 + 移动 + 添加 | ⭐⭐ 中等 |
| **风格不统一** | 拼接后边缘僵硬、光影不一致 | 熔图 或 图生图+参考 | ⭐⭐⭐ 较复杂 |

### 4.2 节点D - 消除与修复（解决简单问题）

**适用情况**：图片中有不需要的元素或小瑕疵

**功能1：消除画笔**
- **操作**：涂抹要消除的区域 → 点击消除
- **要点**：
  - ✅ 宁多勿少（稍微多涂一点）
  - ✅ 完全覆盖目标区域
  - ❌ 不要只涂一半
- **预期效果**：3-5秒后该区域被智能填充

**功能2：细节修复**
- **操作**：点击细节修复按钮
- **适用**：面部扭曲、文字乱码、肢体错误、纹理异常等
- **预期效果**：自动扫描并修复整个画面的微小问题
- **地位**：被称为"万能药"，几乎能解决所有小问题

**典型工作流**：
```
发现问题
    ↓
使用消除画笔移除多余元素（如有）
    ↓
使用细节修复全局优化
    ↓
检查修复效果
    ↓
满意？→ 进入下一环节
不满意？→ 重复或尝试其他方法
```

---

### 4.3 节点E - 重新构图（解决结构性问题）

**适用情况**：图片比例不合适、主体位置不好、想添加新元素

**三步法**：

**Step 1：改变图片比例（扩图）**
- 在画布设置中将比例改为目标尺寸（如1:1→16:9）
- 使用扩图功能自动延展四周空白区域
- 可选：输入描述词引导延展内容
- 从4个候选结果中选择最自然的

**Step 2：移动主体位置（消除+抠图+重排）**
- 使用消除画笔将原主体完全涂抹消除
- 得到一个完整的干净背景图
- 复制原图并对副本执行一键抠图
- 将抠出的主体作为独立图层
- 自由拖动到理想位置（可翻转/缩放/旋转）

**Step 3：添加新元素（可选）**
- 从外部导入新的素材图片
- 对新素材执行一键抠图
- 将其拖入画布，调整大小/位置/角度
- 形成最终的构图布局

**核心价值**：不用重新生成整张图！精确控制每个元素的位置和大小！

---

### 4.4 节点F - 风格统一（解决融合问题）

**适用情况**：多元素拼接后出现风格违和、边缘僵硬、光影不一致

**方案A：熔图（简单快速版）**
- **操作**：将图层叠加 → 使用熔图功能 → 调整强度和羽化参数
- **适用**：简单的边缘过渡问题
- **耗时**：几秒钟
- **效果**：一般（对复杂的光影差异效果有限）

**方案B：图生图 + 图片信息参考模式（高级专业版）** ⭐推荐
- **操作**：
  1. 先将所有元素摆好位置，得到一张"草图"
  2. 使用图生图功能
  3. 选择"图片信息参考模式"
  4. **调整灵魂参数 - 参考强度**：
     - 0.8-0.9：完全保留元素位置
     - 0.5-0.7：平衡保留与风格（**最常用**）
     - 0.2-0.4：大改风格但保留构图
  5. （可选）输入风格提示词："电影质感、光影自然"
  6. 生成4个结果，选择最佳
  7. 最后用细节修复收尾

- **适用**：复杂的风格统一需求、高质量要求
- **耗时**：十几秒到一分钟
- **效果**：优秀（能解决AI多元素崩坏问题）

**决策指南**：
```
需要融合多个元素？
├─ 只是边缘僵硬？ → 用熔图（快）
├─ 光影/色调/风格不一致？ → 用图生图+参考模式（效果好）
├─ 要求快速出图？ → 先试熔图，不行再用图生图
└─ 要求高质量成品？ → 直接用图生图+参考模式
```

---

### 4.5 Phase 2 完成后的决策点

**【人工质检点 #2】**

经过即梦模型的修复后，你应该得到一张**精修后的完美成片**。

**最终检查清单**：

✅ **所有之前的问题都已解决？**  
✅ **没有引入新的问题？**（如过度修复导致的失真）  
✅ **整体风格和谐统一？**  
✅ **画质达到交付标准？**  
✅ **你自己满意吗？（主观感受很重要）**  

**最终决策**：

```
最终检查结果？
    
├─ 完全满意 → 
│   保存导出成品 🎉
│   （可选择进入Phase 3做PS微调，或直接交付）
│
└─ 还是不满意 → 
    分析原因：
    ├─ 小问题 → 返回Phase 2特定节点微调
    ├─ 大问题 → 返回Phase 1重新生成
    └─ 无法解决 → 咨询AI助手寻求替代方案
```

## 八、常见问题快速解决指南

### Q1：生成的图不像本人怎么办？
**快速修复**：
- 降低Denoising Strength至0.4以下
- 在提示词强化："严格保持人物面部特征100%不变"
- 使用参考生图而非纯文生图

### Q2：皮肤看起来很假（塑料感）？
**快速修复**：
- 面部精修改用NanoBananaPro
- 或添加："真实皮肤纹理，保留毛孔细节，避免过度平滑"

### Q3：场景元素不自然？
**快速修复**：
- 调整描述词："少量"、"若隐若现"、"自然融入"
- 尝试不同的场景咒语

### Q4：AI感太强？
**快速修复**：
- 强化超分辨率节点（Phase 1的节点C）
- 添加："降低AI痕迹，模拟真实光学成像"
- 必要时用即梦的细节修复再过一遍

### Q5：处理时间太长？
**优化方案**：
- 中间节点用较低分辨率（最后一步才升分辨率）
- 减少Steps至30-40
- 复杂修复改用单节点快速模式预览
