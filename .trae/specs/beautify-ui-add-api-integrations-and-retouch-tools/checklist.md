# Checklist

## 布局与交互

- [x] 登录后页面呈现左侧工具栏 + 中央大图画布 + 右侧面板的三栏布局
- [x] 左侧工具栏包含：修脸、调色、液化、修复、消除、导出 六个工具入口
- [x] 切换左侧工具时，右侧面板同步切换为对应参数面板
- [x] 屏幕宽度小于 1024px 时，左侧工具栏收缩为图标抽屉，右侧面板变为底部抽屉或标签页
- [x] Header 包含 Provider 选择器和“API 设置”入口
- [x] `useEditor` hook 中新增 `selectedTool` 和 `selectedProvider` 状态，并能正确响应切换

## API Provider 管理

- [x] 共享类型中定义 `ProviderConfig`、`ProviderType`、`ProviderModel` 类型
- [x] 后端实现 Provider CRUD API：`GET /api/providers`、`POST /api/providers`、`PUT /api/providers/:id`、`DELETE /api/providers/:id`
- [x] API Key 在后端加密存储，Provider 列表接口不返回 key 明文
- [x] 启动时从 `GLM_API_KEY` 自动生成默认 GLM Provider，原有 GLM 调用不受影响
- [x] 前端“API 设置”页可添加、编辑、删除、启用/禁用 Provider
- [x] Provider 表单包含名称、类型、API Key、Base URL、默认模型字段
- [x] 前端 Provider 选择器显示所有启用 Provider，切换后编辑请求使用对应 Provider

## 多 Provider 后端代理

- [x] 后端定义统一的 `ImageProvider` 接口
- [x] 现有 GLM 逻辑重构为 `GLMProvider`
- [x] 新增 `OpenAIProvider`，支持 `gpt-image-2` 等模型
- [x] `ProviderFactory` 根据 `providerId` 返回对应 Provider 实现
- [x] `POST /api/edit` 读取请求中的 `providerId` 并正确路由
- [x] Provider 返回 401/403 时前端提示“API Key 无效或已过期”
- [x] Provider 返回 429 时前端提示“该 API 额度已用尽，请切换 Provider 或稍后重试”
