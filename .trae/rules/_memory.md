# 记忆连续性

> **生效模式**：会话启动生效 + 按需生效
> **触发信号**：新会话首次交互，或用户提及历史进度/之前决策
> **优先级**：会话启动时最先执行（在所有其他规则之前）

---

## 触发判定

| 判定条件 | 是否触发 | 示例 |
|---------|---------|------|
| 新会话首次用户消息 | **触发**（启动读取） | 用户在新窗口发送第一条消息 |
| 用户提及"之前"、"上次"、"之前做的" | **触发**（历史检索） | "上次那个飞书配置做到哪了？" |
| 用户询问项目状态/进度 | **触发**（历史检索） | "当前项目进展如何？" |
| 用户表达偏好或决策 | **触发**（写入记忆） | "我以后都用 TypeScript" |
| 纯代码编写/修改 | **不触发** | "把这个函数改成异步的" |
| 简单信息查询 | **不触发** | "Python 的 list comprehension 语法是什么？" |

## 执行逻辑

### 启动读取（新会话首次交互时）

```
新会话首次交互
  ↓
读取 project_memory.md
  ↓
读取最近2天的 topics.md
  ↓
提取与当前消息相关的关键词
  ↓
发现相关历史？
  ├─ 是 → 主动提示："检测到之前的工作：[摘要]，是否延续？"
  └─ 否 → 静默，不干扰用户
```

### 按需检索（用户询问历史时）

```
用户提及历史
  ↓
在 project_memory.md + session_memory 中搜索关键词
  ↓
找到相关信息？
  ├─ 是 → 返回检索结果，附来源文件和时间戳
  └─ 否 → 如实告知"未找到相关记录"，禁止编造
```

### 写入记忆（产生重要信息时）

```
产生重要信息
  ↓
判定信息级别
  ├─ 用户偏好/习惯 → user_profile.md
  ├─ 项目架构/配置变更 → project_memory.md
  └─ 当次会话任务进度 → session_memory_{id}.jsonl
  ↓
写入前：先读取目标文件最新状态
  ↓
写入：追加内容 + 时间戳
```

## 写入触发阈值

满足以下**任一**条件即写入记忆：
1. 用户明确表达偏好（"以后都用..."、"我习惯..."）
2. 项目架构或关键配置发生变更
3. 产生 EXPERIENCE/LESSON/DECISION 类型的经验记录
4. 跨会话需要保持一致的工作上下文

**不写入**：
- 纯代码修改细节（由 git 管理）
- 临时性讨论（未形成决策）
- 通用编程知识（非项目特有）

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| project_memory.md 不存在 | 首次写入时自动创建，使用标准模板 |
| 多窗口同时写入 | 写入前先读取最新状态；若检测到冲突内容，提醒用户确认 |
| 记忆文件过大（>200行） | 提醒用户归档旧条目到 `docs/` 下 |
| 用户否认记忆中的信息 | 以用户当前表述为准，更新记忆 |
| 检索到多条相关记录 | 按时间倒序展示最近 3 条，附时间戳 |
| topics.md 为空或不存在 | 跳过 topics 读取，仅读取 project_memory.md |

## 一致性保障

- **写入前读取**：每次写入前必须先读取目标文件当前内容
- **时间戳**：每条写入必须包含 `YYYY-MM-DD HH:mm` 格式时间戳
- **冲突检测**：若发现同一主题存在矛盾记录，提醒用户确认以哪条为准
- **禁止编造**：检索不到信息时必须如实告知，不得推测或编造
