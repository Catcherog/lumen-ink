# 临时脚本管理

> **生效模式**：智能生效
> **触发信号**：AI 即将创建一次性脚本，或会话中首次涉及 `src/scripts/temp/` 路径
> **优先级**：在 `_file_management.md` 之后执行（先判定目录，再判定是否临时）
> **前置依赖**：若触发此规则，`_file_management.md` 必定已触发

---

## 触发判定

| 判定条件 | 是否触发 | 示例 |
|---------|---------|------|
| AI 创建调试/测试/迁移等一次性脚本 | **触发** | 创建 `debug_api.py` |
| AI 创建项目核心代码文件 | **不触发** | 创建 `src/utils/helper.py` |
| AI 读取 `src/scripts/temp/` 下的文件 | **触发**（巡检） | 扫描过期临时脚本 |
| 用户要求"写个脚本跑一下" | **触发** | 一次性数据处理脚本 |
| AI 创建配置文件或文档 | **不触发** | 创建 `.env` 或 `README.md` |

## 临时脚本判定标准

满足以下**任一**条件即为临时脚本：
1. 用途为调试、测试、数据迁移、格式转换等一次性操作
2. 用户表述含"试一下"、"跑一下"、"临时"等关键词
3. 脚本不需要被其他代码引用

## 执行逻辑

```
判定为临时脚本
  ↓
添加 TEMP 标记 → # TEMP: 用途 | 创建日期 | 预计删除日期
  ↓
存放到 src/scripts/temp/
  ↓
[脚本执行完成]
  ↓
提醒用户：删除 or 保留？
  ↓
删除 → 移除文件
保留 → 迁移至 src/scripts/ + 移除 TEMP 标记
```

## TEMP 标记格式

```
# TEMP: 用途说明 | 创建日期 YYYY-MM-DD | 预计删除日期 YYYY-MM-DD
```

- 预计删除日期默认 = 创建日期 + 3 天
- 用户可指定其他日期

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| 脚本用途不明确 | 询问用户是否为临时脚本，默认按临时处理 |
| 用户要求临时脚本放在其他位置 | 执行但提醒标准位置是 `src/scripts/temp/` |
| 临时脚本后来变成核心代码 | 迁移至 `src/scripts/` 或对应目录，移除 TEMP 标记 |
| `src/scripts/temp/` 目录不存在 | 自动创建后再存放 |

## 巡检逻辑

- **触发时机**：会话中首次涉及文件操作时（与 `_file_management.md` 巡检合并执行）
- **扫描范围**：`src/scripts/temp/` 下所有文件
- **判定规则**：
  - 有 TEMP 标记且超过预计删除日期 → 提醒清理
  - 有 TEMP 标记但未过期 → 不提醒
  - 无 TEMP 标记但文件名匹配 `test_*`、`debug_*`、`temp_*`、`tmp_*` → 提醒确认
- **不执行巡检**：纯对话会话、不涉及文件操作
