# 文件管理规范

> **生效模式**：智能生效
> **触发信号**：AI 即将执行 Write/Edit/Move 等文件操作
> **优先级**：在 `_temp_script.md` 之前执行

---

## 触发判定

| 判定条件 | 是否触发 | 示例 |
|---------|---------|------|
| AI 使用 Write 工具创建新文件 | **触发** | 创建 `src/utils/helper.py` |
| AI 使用 Edit 工具修改现有文件 | **不触发** | 修改 `src/main.py` 中的函数 |
| AI 迁移/重命名文件 | **触发** | 将根目录文件移至 `src/` |
| AI 读取文件 | **不触发** | 读取配置文件内容 |
| 用户要求创建文件 | **触发** | "帮我创建一个测试脚本" |
| AI 在对话中输出代码块但未写入文件 | **不触发** | 在回复中展示代码示例 |

## 执行逻辑

```
文件操作请求
  ↓
确认用途 → 这个文件做什么用？
  ↓
匹配目录 → 代码→src/ 文档→docs/ 资源→public/ 配置→.trae/
  ↓
检查约束 → 根目录？嵌套>3层？跨目录混放？
  ↓
通过 → 执行创建
拒绝 → 提醒用户并建议正确位置
  ↓
创建后验证 → 文件是否在正确目录？
  ↓
位置不当 → 立即迁移至正确位置
```

## 目录分类

| 类型 | 目录 | 示例 |
|------|------|------|
| 代码 | `src/` | .js .ts .py .ps1 .sh |
| 文档 | `docs/` | .md .txt |
| 静态资源 | `public/` | .png .jpg .svg .pdf |
| AI 配置 | `.trae/` | rules/ skills/ knowledge/ |

## 边界情况

| 情况 | 处理方式 |
|------|---------|
| 文件类型不明确（如 .json 配置文件） | 询问用户：是代码配置（`src/config/`）还是文档（`docs/`） |
| 需要在根目录放配置文件 | 仅允许 `README.md` 和构建工具配置（`package.json`、`tsconfig.json` 等） |
| 用户明确要求放在非标准位置 | 执行但记录提醒，写入 project_memory |
| `.trae/` 下的 skill 自带文件 | 不干预，skill 目录结构由 skill 自身管理 |

## 会话巡检

- **触发时机**：会话中首次执行文件操作前
- **执行方式**：快速扫描根目录（仅第一层），发现违规文件时提醒用户
- **不触发情况**：纯对话、纯代码阅读、不涉及文件操作的会话
