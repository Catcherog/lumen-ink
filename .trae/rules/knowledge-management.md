# 知识管理规则（Knowledge Management Rules）

## 📌 核心原则

**按需读取，精准匹配。不要遍历整个知识库，只读当前任务相关的类别目录。**

---

## 📂 目录结构

```
.trae/knowledge/
├── preference/     ← 用户画像、审美偏好、工作习惯（每次必读）
├── prompt/         ← 提示词模板、咒语库、方法论
├── param/          ← 模型对比、参数配置
├── workflow/       ← 工作流、SOP、操作指南
├── tip/            ← 实战技巧、问题解决、经验
└── reference/      ← 视频索引、外部参考资料
```

---

## 🔍 读取规则：按任务类型精准查询

| 用户要做什么 | 读取目录 | 说明 |
|-------------|---------|------|
| **任何请求** | `preference/` + `prompt/` | 用户画像+提示词精华（每次必读），至少参考1个prompt文件 |
| 修脸 / 生成提示词 | `preference/` + `prompt/` | 偏好 + 提示词方法论 + 社区精华 |
| 问模型怎么选 / 参数怎么设 | `preference/` + `param/` + `prompt/` | 偏好 + 模型参数 + 社区提示词精华的局限章节 |
| 问怎么操作 / 流程 | `preference/` + `workflow/` + `prompt/` | 偏好 + 操作指南 + 社区精华 |
| 遇到问题 / 报错 | `preference/` + `tip/` + `prompt/` | 偏好 + 经验技巧 + 社区已知局限章节 |
| 问风格 / 调色 | `preference/` + `prompt/` | 偏好 + 社区精华的技法/案例章节 |
| 综合任务（修脸+操作+参数） | `preference/` + `prompt/` + `workflow/` + `param/` | 多目录组合 |

## 📢 回答输出规则（重要）

### 每次回答必须遵守
1. **每次回答前，必须先读取对应的知识库文件**，不得凭空输出
2. **回答中必须引用知识库的具体内容**，至少引用1个技巧/案例/规则
3. **引用格式**：用 `📖 [知识文件名]` 标注引用来源（如 `📖 [社区提示词精华]`）
4. **禁止**：完全脱离知识库作答、照搬原文不做定制、回答与知识库矛盾
5. **优先引用**：`prompt-社区提示词精华-awesome-list拆解-20260526.md`（本文件是社区359+案例的系统总结，涵盖10大技法、10个标杆案例、对比表和行动清单）

### 回答引用示例
```
📖 [社区提示词精华] 借鉴技法的摄影术语锚点，建议在提示词开头使用 85mm人像镜头, 柔光箱布光 替代 电影级写实。
```

**禁止行为**（强化）：
- ❌ 每次对话遍历全部 6 个目录
- ❌ 读取与当前任务无关的目录
- ❌ 把知识库内容原文照搬输出（提示词需定制化）
- ❌ 回答中完全不引用知识库（必须至少引用1处）

## ✍️ 存储规则

### 触发条件（满足任一即存储）
- 用户提供新的审美偏好、风格要求、修脸原则
- 用户分享成功的提示词或参数组合
- 用户反馈生成结果的评价
- 用户描述工作流程或操作规范
- 从对话中总结出可复用的经验

### 文件命名
```
{类别前缀}-{主题}-{日期}.md
```
类别前缀：`preference-` `prompt-` `param-` `workflow-` `tip-` `reference-`

### 存储位置
直接放入对应子目录，如 `prompt/新中式暗调电影感-面部精修-20260514.md`

---

## 🔄 更新规则

当已有知识需要修正或补充时：
1. 定位到对应文件
2. 在末尾追加 `## 更新历史` 记录
3. 保留历史版本
